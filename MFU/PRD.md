# 产品需求文档 (PRD)

## 1. 产品概述
**项目名称**：人体筋膜与穴位交互式3D检索平台
**项目目标**：打造一个基于 Web 的结构化解剖可视化工具。用户可以通过点击直观的3D人体模型，或通过检索系统，双向联动地快速查找、定位肌肉/筋膜以及其关联的穴位信息。
**核心理念**：遵循“简洁至上”原则，摒弃所有多余设计，专注于“视觉定位 (左)”与“信息知识 (右)”的无缝联动，为专业人士或爱好者提供直观且无缝的浏览体验。

## 2. 界面布局层级
页面采用现代应用的 **双栏自适应布局 (Two-column Split Layout)**，在宽屏 PC 设备上呈现为左右分布：
*   **左区 (视觉交互层 | 占比约 50%-60%)**：承载3D立体解剖模型。
*   **右区 (信息检索层 | 占比约 40%-50%)**：承载高阶搜索框及信息结果卡片。

## 3. 核心功能需求

### 3.1 左侧功能：三维互动模型
1.  **3D模型渲染与操作**：
    *   加载高精度的肌肉/筋膜三维立体模型。
    *   支持鼠标全指向操作：拖拽实现 360° 旋转，滚轮实现缩放 (Zoom-in/out)，右键实现平移。
2.  **部位悬停与拾取**：
    *   **悬停 (Hover) 响应**：鼠标经过特定肌肉组织时提供呼吸灯级别的高亮提示，并可展示轻量化的 Tooltip (肌肉名称)。
    *   **点击选定**：点击操作选定目标肌肉后，触发右侧信息面板的数据查询及加载事件。

### 3.2 右侧功能：词条搜索与内容展示
1.  **全局检索 (Search Bar)**：
    *   位于面板顶部，支持输入肌肉名称、筋膜区域或穴位名称进行模糊检索。
    *   输入时支持下拉联想选词 (Auto-complete)。
2.  **详情展示卡片 (Detail Cards)**：
    *   采取整洁的模块化设计（手风琴折叠面板 等）。
    *   **肌肉详情区**：展示选定肌肉的名称、功能描述、关联筋膜链等。
    *   **关联穴位列表**：展示附着或位于该肌肉区域的穴位集合，条目可展开查看具体主治、进针方式、定位解剖图等文本信息。

### 3.3 核心亮点：双向状态联动
1.  **左指右展（图形查信息）**：用户在左侧 3D 模型直观点击某块肌肉片段时，右侧列表立即呈骨架屏加载，并平滑过渡渲染出该片段的详细解剖数据与穴位清单。
2.  **右搜左引（检索定位图）**：用户在右侧搜索栏主动搜索到某个肌肉或穴位并回车后，左侧的 3D 摄像机 (Camera) 自动执行平滑的漫游动画，将视角聚焦特写至该部位，并施予高亮效果。

## 4. 非功能性需求与性能要求
1.  **性能极客**：考虑到 3D 模型的高多边形体量，必须提供模型懒加载 (Lazy Load)、加载进度条展示。
2.  **自适应设计**：以 PC 端横屏为主进行布局优化。如遇屏幕较小的移动端设备，应优雅降级为“上图下文”的堆叠式布局。
3.  **视觉规范**：采用医疗/健康科技感的主题配色（建议暗黑模式或者偏向生理解剖类的极简风格）。
